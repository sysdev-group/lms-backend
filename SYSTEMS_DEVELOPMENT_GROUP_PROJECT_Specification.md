# SYSTEMS DEVELOPMENT – GROUP PROJECT SPECIFICATION

**Module Code: UFCF7S- 30 - 2_Sep_
Academic Year: 2025/**

**Introduction**

Each group (4 students) will design, develop, and evaluate a web-based Learning Management
System (LMS).

The system must demonstrate application of:

- Software Development Life Cycle (SDLC)
- Requirements Engineering
- UML Modelling
- Database Design
- Risk Management
- Ethical & Legal Considerations
- Testing and Quality Assurance

The final system must be fully functional and demonstrate clear improvement over a standard
Moodle-style platform.

**Project Aim**

To design and implement an enhanced Moodle-like LMS system that improves usability,
performance, and user experience while ensuring security, ethical compliance, and professional
software engineering standards.

**Deliverables**

The group project consists of:

```
Draft Specification
```
- System Overview
- Functional & Non-functional Requirements


- Initial UML diagrams
- Development methodology selection
- Risk assessment

```
Final Submission
Each group must submit:
```

- Source code (GitHub repository or ZIP file)
- UML diagrams (Use Case, Class, Sequence, Activity)
- ER Diagram
- Test Plan document (minimum 3 detailed test cases)
- Brief system documentation
- Live demonstration
